## Theme Application Rules

1. **Default theme**: Corporate Dark (unless user specifies otherwise)
2. **Theme selection**: If user says "dark" → Neon. "Minimal" → Monochrome. "Warm" → Warm Sunset. "Blue/cool" → Ocean Breeze. "Professional" → Corporate Dark.
3. **Shadow**: Enable `shadow=1` on outer frame and cards for all themes except Monochrome
4. **Rounded corners**: All cards use `rounded=1;arcSize=4-8` (smaller = larger containers, bigger = smaller elements)
5. **Font**: Use `fontFamily=Helvetica` for labels, `fontFamily=Courier New` for code/paths
6. **Differentiation**: When a diagram has 3+ peer elements, assign each a different card color from the theme
7. **Consistency**: Within a section, all elements of the same semantic type share the same card color
